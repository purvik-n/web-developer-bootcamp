1. Document Structure

Every HTML document must begin with a proper boilerplate:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title</title>
</head>
<body>
    <!-- Content goes here -->
</body>
</html>
<!DOCTYPE html>
Tells the browser this is an HTML5 document. Always the first line.
<html lang="en">
Root element; lang attribute helps screen readers and search engines.
<head>
Contains metadata — not visible to the user.
<meta charset="UTF-8">
Sets character encoding so special characters display correctly.
<body>
Contains all visible page content.

2. Headings & Text Formatting

HTML provides six levels of headings, from h1 (most important) to h6 (least important). There should only be one h1 per page.

Text Emphasis Tags

<em> Emphasis (semantic) Italic text
<i> Italic (stylistic only) Italic text
<mark> Highlighted text Highlighted
<sup> Superscript x2
<sub> Subscript H2O

3. Links & Anchors

The anchor tag <a> creates hyperlinks. Key attributes:

  • href — destination URL (required)
  • target="_blank" — opens in a new tab
  • rel="noopener noreferrer" — security best practice for _blank links
  • title — tooltip text on hover

Types of URLs

Type Example Use Case
Absolute URL https://www.google.com Linking to external websites
Relative URL about.html Linking between pages in the same project
Anchor (fragment) link #section-id Jumping to a section on the same page
Email link mailto:me@example.com Opens the user's email client

4. Images

The <img> tag is a void/self-closing element (no closing tag).

<img src="photo.jpg" alt="A descriptive text" width="400" height="300">

Key Attributes

  • src — path to the image file (required)
  • alt — alternative text for accessibility and SEO (required)
  • width / height — dimensions in pixels
  • loading="lazy" — defers loading until the image is near the viewport

Rule: Always include a meaningful alt attribute. Empty alt="" is acceptable for purely decorative images.

5. Lists

Unordered Lists <ul>

Use for items with no particular order:

  • Apples
  • Bananas
  • Cherries

Ordered Lists <ol>

Use for sequential steps or ranked items:

  1. Open the terminal
  2. Navigate to your project folder
  3. Run git init

Description Lists <dl>

Use for term–definition pairs (like a glossary):

HTML
HyperText Markup Language — the standard language for web pages.
CSS
Cascading Style Sheets — used to style HTML elements.
JavaScript
A scripting language that adds interactivity to web pages.

6. Tables

HTML tables are for tabular data, not for page layout.

Table Structure

HTML Table Elements Reference
Element Description
<table> Container for the entire table
<caption> Title or description of the table (placed right after <table>)
<thead> Groups the header rows
<tbody> Groups the body rows
<tfoot> Groups the footer rows (totals, summaries)
<tr> Table row
<th> Table header cell (bold & centered by default)
<td> Table data cell
colspan Makes a cell span multiple columns
rowspan Makes a cell span multiple rows

7. Semantic HTML5 Elements

Semantic elements communicate the role of the content to the browser and assistive technologies. Always prefer semantic elements over generic <div> wrappers.

HTML5 Semantic Element Reference
Element Purpose
<header> Introductory content or navigational links for a page/section
<nav> Block of navigation links
<main> Dominant content of the page — only one per page
<section> Thematic grouping of content, usually with a heading
<article> Self-contained content that can stand alone (blog post, news article)
<aside> Content tangentially related to the surrounding content (sidebars)
<footer> Footer for a page or section
<figure> Self-contained content like images, diagrams, code snippets
<figcaption> Caption for a <figure> element

8. HTML Entities

Special characters must be escaped using HTML entities to display correctly and avoid breaking the markup parser.

Common HTML Entities
Character Entity Description
< &lt; Less than sign
> &gt; Greater than sign
& &amp; Ampersand
© &copy; Copyright symbol
™ &trade; Trademark symbol
— &mdash; Em dash
  &nbsp; Non-breaking space

9. Key Takeaways

  1. HTML is the skeleton of every web page — structure first, style later (with CSS).
  2. Always use semantic tags — they improve accessibility, SEO, and readability.
  3. The alt attribute on <img> is not optional — it's essential.
  4. Relative URLs keep your project portable; absolute URLs link to external resources.
  5. A proper heading hierarchy (h1 → h6) helps both users and search engines.
  6. HTML entities prevent your markup from breaking when displaying special characters.
  7. Tables are for data — do not use them for layout purposes.

4. Form Elements Reference

HTML Form Input Types — Quick Reference
Input Type Syntax Purpose
text type="text" Single-line plain text
email type="email" Email address — browser validates format
password type="password" Masked text input
number type="number" Numeric input with min/max/step
tel type="tel" Phone number (triggers numeric keyboard on mobile)
url type="url" URL input — browser validates "https://" format
date type="date" Date picker — returns YYYY-MM-DD
radio type="radio" Single select from a group (same name)
checkbox type="checkbox" Multi-select toggle inputs
range type="range" Slider for numeric values between min/max
submit type="submit" Submits the form
reset type="reset" Resets all inputs to their default values

5. Key Takeaways on HTML Forms

  1. Always pair every <input> with a <label> using matching for / id attributes — essential for accessibility.
  2. The name attribute on an input is what gets sent to the server — without it, the value is ignored on form submit.
  3. Use <fieldset> + <legend> to group related inputs (especially radio/checkbox groups).
  4. HTML5 provides built-in validation via required, pattern, minlength, min, max — no JavaScript needed for basics.
  5. method="get" appends data to URL (for search/filter forms); method="post" sends data in the request body (for sensitive/large data).
